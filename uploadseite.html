Uploadseite

Erstellung einer Uploadseite zur Einpflegung des Speiseplans

Problem

In der Betriebskantine werden auf einem Display der aktuelle Speiseplan und Werbefolien angezeigt. Der Speiseplan für die nächste Woche kommt immer per E-Mail in das IT-Gruppenpostfach. Ursprünglich musste man, damit der neue Speiseplan und die Werbefolien auf das Display kommen, die PDF-Dateien von Hand in ein JPG umwandeln, die Dateien auf einen Server legen und die Dateinamen im Quellcode ändern. Dieser Vorgang soll vereinfacht werden.

Ziel / Lösung

Mit der Uploadseite kann man die PDF-Datei auf den Server hochladen und muss sich um nichts Weiteres mehr kümmern. Sie wird automatisch zu einem JPG umgewandelt und in den richtigen Ordner verschoben. Der alte Speiseplan wird automatisch am Ende der Woche durch den neuen ersetzt. Jeder hochgeladenen Speiseplan wird mit Dateiname und Kalenderwoche, für die der Speiseplan gilt, in einer Datenbank gespeichert. Dadurch kann man auch mehrere Speisepläne für die kommende Wochen planen. Ein Bash-Skript, das per Cronjob regelmäßig ausgeführt wird, sucht aus der Datenbank den Dateinamen, des neuen Speiseplans, der angezeigt werden soll, raus und verschiebt den Speiseplan ins Zielverzeichnis. Bei den Werbefolien wird es genauso gemacht. Hier kann man aber zusätzlich noch das Start- und das Enddatum der Werbefolien angeben. Das Bashskript bringt am Startdatum die Werbefolie automatisch auf das Display und entfernt sie am Enddatum automatisch wieder.

Programmiersprachen / Techniken

HTML, CSS, PHP, Bash, SQL
